# Security Guardrails (The Bunker's Constitution)

These guardrails define the non-negotiable security boundaries for all agents and developers operating within the Bunker. They are enforced by [[OzyZT]] and verified by [[Trivy]].

## 🛡️ The 5 Pillars of Bunker Security

### 1. Secret Governance
- **Zero Secrets in Code**: No API keys, passwords, or tokens in the repository. Ever.
- **Verification**: Mandatory pre-commit hooks (using `gitleaks` or `trufflehog`) to scan for entropy and known patterns.
- **Storage**: All secrets must live in an encrypted vault (Vault/ECR) and injected at runtime via environment variables.

### 2. Supply Chain Integrity
- **Lockfile Pinning**: Always commit `package-lock.json`, `go.sum`, etc.
- **Dependency Auditing**: Automated `npm audit` or `go list -m all` on every build.
- **Provenance**: Use **Trusted Publishing** (OIDC) for all deployments. No long-lived tokens.

### 3. Least Privilege Architecture
- **Service Isolation**: Each service must have its own DB user with the minimum required permissions.
- **Container Hardening**: Run as non-root, use read-only filesystems where possible.
- **Network Zoning**: Services in the same cluster cannot talk to each other unless explicitly allowed by a network policy.

### 4. Offensive Vaccine (Active Auditing)
- **Continuous Scanning**: [[Trivy]] runs every 24 hours on all active images.
- **Drift Detection**: Automated checks to ensure the production environment hasn't deviated from the IAC (Infrastructure as Code) definition.
- **Payload Verification**: Use [[PayloadsAllTheThings]] patterns to test inputs during integration tests.

### 5. AI Agent Guardrails
- **Prompt Isolation**: Agents must not have access to raw secret values.
- **Output Sanitization**: All code generated by AI must pass through [[react-doctor]] and a security linter before being merged.
- **Human in the Loop**: Critical infrastructure changes require manual approval.

## 🚦 Security Gates (OzyZT Integration)

| Level | Action | Tooling |
|-------|--------|---------|
| **P0 (Critical)** | Block Commit | Secret Detection, Syntax Errors |
| **P1 (High)** | Block PR | Vulnerability Scan, Linting |
| **P2 (Medium)** | Warn in PR | Style violations, Deprecated APIs |
| **P3 (Low)** | Background Task | Dependency updates, Documentation gaps |

## ⚠️ Violation Protocol
1. **Detection**: Gate triggers non-zero exit code.
2. **Alert**: Notification sent to Slack and logged in `wiki/log.md`.
3. **Quarantine**: The affected branch or image is blocked from moving forward.
4. **Resolution**: Mandatory fix before the gate can be cleared.
